OO Address Book Text

As we start to make programs that go beyond a single method, they'd quickly become unmanageably complex if we didn't have a good way to organize our code. The most common paradigm used to organize software these days is object-oriented design, or OO. OO uses these things called objects to organize code. Objects do many things, but we'll start with the most simple: they store information. For example, we can use some objects to store information about famous people in the Chrome JavaScript console:

var person1 = { title: "Dr.", lastName: "Seuss" };
            var person2 = { title: "President", lastName: "Nixon" };
            var person3 = { title: "Mr.", lastName: "Rogers" };

The title and lastName are called properties, and the information they store is called values. The variable name doesn't actually matter, except to make your code readable.

Since all of our people have the same properties, we can create a function that any of them can be passed into:

var sayHello = function(person) {
              alert("Hello, " + person.title + " " + person.lastName + "!");
            };

Often, we'll want to work with a collection of similar objects, so we'll create an array to store our contacts in:

var people = [person1, person3, person3];
            people.forEach(function(person) {
              sayHello(person);
            });

Let's check out another example. We'll make some objects to represent animals and the sounds they make:

var cow = { name: 'cow', sound: 'moo' };
            var horse = { name: 'horse', sound: 'neigh' };
            var fox = { name: 'fox', sound: '???' };

            var farmAnimals = [cow, horse, fox];

            var makeNoise = function(animal) {
              console.log('The ' + animal.name + ' says ' + animal.sound + '.');
            };

            farmAnimals.forEach(function(animal) {
              makeNoise(animal);
            });

Now that we know how to make objects, let's work on incorporating them into a web page. We'll start out with this code:

scripts.js
$(document).ready(function() {
              $("form#new-contact").submit(function(event) {
                event.preventDefault();

                var inputtedFirstName = $("input#new-first-name").val();
                var inputtedLastName = $("input#new-last-name").val();
                var inputtedAddress = $("input#new-address").val();
                var newContact = { firstName: inputtedFirstName, lastName: inputtedLastName, address: inputtedAddress };

                $("ul#contacts").append("<li><span class='contact'>" + newContact.firstName + " " + newContact.lastName + "</span></li>");

                $("input#new-first-name").val("");
                $("input#new-last-name").val("");
                $("input#new-address").val("");
              });
            });

Now, if we fire up our web page, each time we add a contact, it's added to the list.

This isn't very exciting. We could have done this all with jQuery and not mucked around with objects. Let's make our page more complex to show off the power of objects. We'll add a feature where if a user clicks a contact, they will be able to see the full information on the contact.

First, we'll update our HTML, and add the following code at the end of our <div class="row">:

address-book.html
<div class="col-md-6">
              <div id="show-contact">
                <h2></h2>

                <p>First name: <span class="first-name"></span></p>
                <p>Last name: <span class="last-name"></span></p>
                <p>Address: <span class="address"></span></p>
              </div>
            </div>

And we'll hide our contact's "show" area at first, and add a class to make some elements look clickable:

styles.css
#show-contact {
              display: none;
            }

            .contact {
              cursor: pointer;
              color: #0088cc;
            }

            .contact:hover {
              text-decoration: underline;
            }

Now, let's update our JavaScript, and add this to the end of our form submit callback:

scripts.js
$(".contact").last().click(function() {
              $("#show-contact").show();
              $("#show-contact h2").text(newContact.firstName + " " + newContact.lastName);
              $(".first-name").text(newContact.firstName);
              $(".last-name").text(newContact.lastName);
              $(".address").text(newContact.address);
            });

Can you imagine trying to keep track of everything on this page using just jQuery? Hopefully you're starting to see the power of object-oriented design now.
